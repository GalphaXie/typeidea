### 1.commit 文档提交规范
参考: https://www.ruanyifeng.com/blog/2016/01/commit_message_change_log.html
```shell script
<type>(<scope>): <subject>
// 空一行
<body>
// 空一行
<footer>
```
#### type 标识符
>- feat：新功能（feature）
>- fix：修补bug
>- docs：文档（documentation）
>- style： 格式（不影响代码运行的变动）
>- refactor：重构（即不是新增功能，也不是修改bug的代码变动）
>- test：增加测试
>- chore：构建过程或辅助工具的变动
